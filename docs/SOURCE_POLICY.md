# 情報源ポリシー

## 登録条件

1. 公式発表、公式RSS/Atom、利用許諾されたAPIを優先
2. 公開者、URL、取得日時を保存できる
3. 本文の再配布を目的にしない
4. ログイン・有料購読を回避しない
5. 取得頻度を抑え、キャッシュする
6. 情報訂正を検知できる

## 信頼レベル

- 5: 官公庁、メーカー・サービス公式
- 4: 業界団体、上場企業IR、公式PR
- 3: 編集責任者が明確な報道媒体
- 2: 一般ブログ、まとめ
- 1: 出所不明、SNS単独情報

自動記事生成に使う一次根拠は原則4以上。レベル3以下は流行検知の補助に限定し、公式ソースが見つかるまで公開しません。

## 国内と海外のトレンド表記

- 日本・アメリカ・韓国・イギリスのGoogle Trendsを別レーンで監視する
- 日本の現在の注目と海外の検索急上昇を最低1件ずつ候補化する
- 国名、情報源、検索規模、確認日を紹介文に残す
- Google Trendsは「検索急上昇」、Google Newsは「ニュースで注目」と書く
- SNS上の投稿量を確認していない場合は「SNSでバズ」と書かない
- 海外の話題と日本の楽天商品が同一とは限らない場合、その旨を明記する
- 日本で購入可能な関連商品と照合できた話題だけを商品導線へ進める

## 気象・災害

気象庁は防災情報XMLをユーザー登録不要のPULL型で提供しています。ただし、警報や災害発生中の商品販売は不安を利用しやすいため、初期設定では取得候補を無効にしています。将来有効化しても、災害記事は販売導線を外し、平時の防災ガイド更新だけに利用します。

## 候補台帳

`config/source_candidates.json` は調査候補です。`active: false` のままではBOTは取得しません。利用条件・フィード形式・内容をCEOまたは品質部が確認後、`config/sources.json` へ移します。
