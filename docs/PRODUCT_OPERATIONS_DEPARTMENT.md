# 商品運用部BOT

## 目的

掲載商品を固定せず、販売状況・価格・レビュー・需要適合・成果を継続監視し、サイトの商品棚を自動更新する。
社長は個別商品のスクリーニングを行わず、例外と月次方針だけを確認する。

## 組織

- 商品運用部長BOT: 更新ジョブの統括、判断履歴、CEO報告
- 商品監視BOT: 販売終了、リンク、価格、レビュー平均・件数を日次確認
- 商品選定BOT: 楽天APIから商品枠ごとの候補を収集・採点
- 商品入替BOT: 隔週で現商品と新候補を比較し、基準を満たす場合だけ差し替え
- 品質監査BOT: 商品タイプ不一致、低評価、レビュー不足、頻繁な入替を防止
- 分析BOT: クリック、成果、売上データが入った後に商品別利益へ反映

## 運用周期

### 毎日

販売状況、価格、レビュー、アフィリエイトURLを確認する。
APIの一時障害で商品を消さないため、販売確認が2回連続で失敗した場合のみ代替候補を探す。

```bash
python3 -m trend_commerce product-ops --mode daily
```

### 2週間ごと

各商品枠を再スクリーニングする。候補はレビュー20件以上、平均4.0以上、選定75点以上を基本条件とする。
通常商品は最低12日掲載し、候補の総合指数が現商品を6点以上上回る場合だけ差し替える。

```bash
python3 -m trend_commerce product-ops --mode rotation
```

## 判断履歴

- DB: `product_operation_runs`, `product_decisions`, `product_slot_state`
- JSON: `output/product_operations/`
- 公開サイトには運営判断や分析画面を出さない

## 安全装置

- 商品タイプ不一致の候補は自動採用しない
- レビュー20件未満・平均4.0未満は通常採用しない
- API失敗1回では商品を差し替えない
- 最低掲載期間を設け、商品が毎回変わる状態を防ぐ
- 全判断に理由と変更前後URLを保存する
