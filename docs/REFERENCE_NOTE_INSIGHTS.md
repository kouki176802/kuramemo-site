# 参考noteから取り込むノウハウ

作成日: 2026-06-25

参考URL: https://note.com/09pauai/n/n5caeb9365cb9

## 前提

参考記事は、SNSアフィリエイトで成果を出すための考え方として「フォロワー数より見込み客の質」「ポスト→信頼構築→販売」「SNS共通の心理導線」を強調している。

ただし、強い煽り、過度な収益実績訴求、限定性の強調、攻撃的表現はそのまま採用しない。選び方メモでは、比較メディアとしての信頼を優先し、安全な形に変換して使う。

## 取り込む考え方

### 1. フォロワー数より見込み客の質

フォロワーを数で増やすより、商品・比較ページに関心がある人を集める。

選び方メモでの運用:

- 相互フォロー目的の運用をしない
- 投稿ごとに「誰向けか」を明確にする
- 比較ページに来る理由がある投稿を作る
- 伸びた投稿でも、購買意図が薄いテーマは増やしすぎない

BOTへの反映:

- インプレッションだけでなく、リンククリック率、保存率、プロフィール遷移を重視
- フォロワー増より、比較ページ流入を評価

### 1.5 売れないNG行動

参考資料で繰り返し出ていたNGは、選び方メモでもそのまま重要。

避けること:

- フォロワー数だけを追う
- 信頼前にリンクを貼る
- 情報だけを並べて個性がない
- 発信テーマと関係ない案件を紹介する
- 低単価だけで消耗する
- 全員向けに書く
- 完璧主義で公開が遅れる

選び方メモでの変換:

- 数より「比較ページに来る見込み客」を重視
- 初期は広告リンクより比較・信頼投稿を優先
- 体験談がない場合は、編集部の判断過程・比較基準を出す
- 案件はカテゴリ文脈とつながるものだけ扱う
- 物販だけに依存せず、SaaSや高単価案件も将来候補に入れる
- 投稿は1人の読者に向けて書く
- 70%で公開し、反応を見て直す

### 2. 毎日投稿より、正しい導線

投稿量だけ増やしても、導線が弱いと成果につながらない。

選び方メモでの導線:

```text
悩み・話題
↓
買う前チェック
↓
比較ページ
↓
個別チェック
↓
アフィリエイトリンク
```

BOTへの反映:

- 投稿文には必ず「比較ページへ送る理由」を入れる
- 商品単体紹介より、選び方・チェックリスト・ランキングを優先
- SNS投稿は記事の宣伝ではなく、読者の判断負荷を下げる入口にする

### 3. 信頼構築が先、販売は後

売る前に、読者に「このアカウントは判断材料になる」と思ってもらう必要がある。

選び方メモでの信頼要素:

- 買わなくていい人を書く
- 代用品を書く
- 注意点を書く
- 広告表記を隠さない
- 未使用商品を体験談にしない
- 比較基準を明記する

BOTへの反映:

- 投稿タイプを「信頼投稿」「比較投稿」「販売補助投稿」に分ける
- 初期は販売補助投稿より、信頼投稿と比較投稿を多めにする

### 4. SNS共通の心理導線

プラットフォームが変わっても、人が買うまでの心理は大きく変わらない。

選び方メモでの共通導線:

1. それ、自分に関係あるかもと思う
2. 買う前に失敗を避けたいと思う
3. 比較基準を知る
4. 自分向けの選択肢を見つける
5. 条件を確認する
6. 納得してリンク先へ進む

### 5. リプ・コメント導線

参考記事では、大きいアカウントへのリプや会話参加も重要視されている。

選び方メモでの安全な使い方:

- 宣伝リプをしない
- 相手の投稿に対して、短く役立つ補足をする
- 比較ページのリンクをむやみに貼らない
- 返信は「売る」より「信頼を作る」目的にする

例:

- 「それ選ぶなら、端末側の対応W数も見た方が安全です」
- 「暑さ対策系は、外出用と就寝用で見るポイントがかなり違います」
- 「AIツールは、有料化前に解約条件と商用利用を見たいですね」

BOTへの反映:

- 将来「返信案生成」を追加する
- 返信案にはリンクを原則入れない
- 宣伝、煽り、絡み目的の返信は禁止

### 6. 教育要素の分解

参考記事では、販売までに必要な教育要素として、目的、信用、問題点、手段、投資、行動の順序が示されている。

選び方メモでは、これを次のように置き換える。

| 要素 | 選び方メモでの使い方 |
|---|---|
| 目的 | 何のために選ぶのか |
| 信用 | 比較基準、広告表記、買わない選択肢 |
| 問題点 | 失敗しやすい選び方 |
| 手段 | 比較表、チェックリスト、ランキング |
| 投資 | 価格、維持費、解約、返品条件 |
| 行動 | リンク先で最新条件を確認 |

## DRM 6ステップの採用方針

参考資料の中心は、売る前に読者を順番に教育すること。

選び方メモでは、攻めた販売色を弱めて、次の6段階に変換する。

### 1. 目的

読者に「何を避けたいか」「どう選べば楽になるか」を見せる。

例:

- 充電器選びで失敗しない
- AIツールの固定費を増やさない
- スキンケアを買いすぎない

### 2. 信用

広告より先に、比較基準と注意点を出す。

例:

- 買わなくていい人を書く
- 代用品を書く
- 価格・保証・解約条件を見る

### 3. 問題点

読者の失敗パターンを言語化する。

例:

- ワット数だけで充電器を選ぶ
- 大容量だけでモバイルバッテリーを選ぶ
- SNSで見たスキンケアを一気に買う

### 4. 手段

比較ページ、ランキング、チェックリストを提示する。

### 5. 投資

価格だけでなく、維持費、解約、返品、手間を確認させる。

### 6. 行動

「買う」ではなく、まずリンク先で最新条件を確認する行動へ促す。

## 初期アカウント運用の採用方針

参考資料では「信頼前にリンクを貼らない」ことが強調されていた。

選び方メモでも、初期は以下にする。

- 1〜2週間目: リンクなし投稿中心
- 3〜4週間目: 比較ページ誘導を少し入れる
- サイト公開後: 比較ページへのリンクを増やす
- ASP承認後: 広告リンク付き投稿を限定的に開始

商品リンクの直接投稿は、最後でよい。

## ストーリー要素の扱い

参考資料では、情報よりストーリーが反応されやすいとされていた。

ただし選び方メモは、実体験が薄い商品を体験談化しない方針。

安全なストーリー化:

- 「過去の自分ならここを見る」
- 「買う前に迷いやすいポイント」
- 「よくある失敗」
- 「比較して気づいた注意点」
- 「SNSで話題だけど、先に確認したいこと」

使わない:

- 架空の体験談
- 知り合いが使った風の話
- 実績や効果の誇張
- 収益・健康・美容効果の保証

## 高単価案件について

参考資料では高単価案件の重要性が強く書かれていた。

選び方メモでは、初期は物販で媒体を作り、将来は以下のような高単価寄り案件を検討する。

- AI/SaaS
- オンラインスクール
- 転職・キャリア
- 美容サービス
- 通信・セキュリティ

ただし、金融、投資、医療、美容医療、保険、不動産は規制・表現リスクが高い。自動収益化せず、扱う場合は別ルールとCEO確認を必須にする。

## 70%公開・改善主義

参考資料の「完璧主義をやめる」は採用する。

選び方メモでの実行:

- 記事は公開前100点を狙わない
- 比較基準、広告表記、禁止表現だけ守る
- 初期は下書き・手動投稿で出す
- 反応を見てタイトル、導入、比較表、CTAを改善する
- 毎週、伸びた投稿を比較ページに反映する

## 取り込まないもの

以下は採用しない。

- 過度な収益実績アピール
- 強い焦り訴求
- 他者を見下す表現
- 期限や値上げを使った煽り
- 「誰でも簡単に稼げる」表現
- 大量アカウント運用によるスパム的展開
- コピペ投稿
- 独自フレーズの丸パクリ

## BOTに追加したい機能候補

### 1. 投稿ステージ分類

投稿を次の4種類に分ける。

- 認知投稿: 話題に気づかせる
- 信頼投稿: 買わない選択肢、注意点、比較基準
- 比較投稿: ランキング、比較表、用途別
- 行動投稿: 比較ページやチェック記事へ誘導

### 2. 見込み客スコア

投稿テーマごとに、単なる話題性ではなく購買意図を評価する。

見る指標:

- 保存率
- リンククリック率
- プロフィールクリック率
- 比較ページ滞在
- アフィリエイトクリック

### 3. 返信案生成

XやThreadsで、宣伝ではない有益な返信案を作る。

ルール:

- リンクは原則なし
- 相手を否定しない
- 短い補足
- 商品名を押し付けない
- 広告目的の返信にしない

### 4. 月別案件カレンダー

季節需要を先回りする。

例:

- 1月: 新生活準備、家トレ、スキンケア
- 3月: 新生活、ガジェット、収納
- 5月: 暑さ対策、日焼け止め
- 6月: 梅雨、除湿、部屋干し
- 11月: 防寒、ブラックフライデー
- 12月: 年末整理、ギフト、セール

## 初期運用への反映

今後のSNS投稿比率:

- 認知投稿: 30%
- 信頼投稿: 30%
- 比較投稿: 30%
- 行動投稿: 10%

初期は売り込みを強めない。比較ページと信頼構築を先に育てる。
