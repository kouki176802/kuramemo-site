# 投稿BOT運用ルール

作成日: 2026-06-25

## 目的

投稿BOTは、トレンドやニュースを起点に、SNS投稿、ブログ記事、アフィリエイト導線を半自動で作る中核システム。

ただし初期は完全自動公開しない。まずは下書き生成、重複防止、予約キュー、dry-run、CEO確認を中心に運用する。

## 運用レベル

### Level 0: 設計

- 外部投稿なし
- API課金なし
- ローカルでテンプレートとルールを作る

### Level 1: Shadow Mode

- 記事とSNS投稿を自動生成
- 投稿はdry-run
- アフィリエイトリンクは仮リンク
- CEOが品質確認

現在の状態: Level 1

### Level 2: 手動公開

- BOTが投稿案を作る
- CEOが確認して手動投稿
- ASP審査用の実績を作る

### Level 3: 半自動公開

- 低リスク投稿のみ予約投稿
- 収益記事、健康、美容、広告リンク付き投稿はCEO確認

### Level 4: 自動公開

- 実績のあるカテゴリのみ自動公開
- エラー、否認率、炎上リスク、規約変更を監視
- CEOは週次レビュー

## 投稿BOTの担当範囲

### 収集担当

- RSS/Atomを取得
- 公式情報源を優先
- 重複URLを除外
- 同じ話題をクラスタ化

### 商機評価担当

- 話題性
- 購買意欲
- 季節性
- 商品との近さ
- SNS化しやすさ
- リスク

### 記事担当

- 集客記事
- 比較記事
- 収益記事
- タイトル
- メタディスクリプション
- FAQ
- 広告表記

### SNS担当

- X投稿文
- Threads投稿文
- Instagramカルーセル文
- TikTok台本
- 投稿予約
- 重複投稿防止

### 収益化担当

- 商品候補を探す
- ASP案件と照合
- アフィリエイトリンク挿入候補を作る
- 未承認案件は仮リンクにする

### コンプライアンス担当

- ステマ表記確認
- 未使用商品の体験表現を検出
- 健康・金融・災害・事件事故の自動収益化を止める
- 煽りすぎ、偽の期限、架空口コミを止める

### 分析担当

- 投稿履歴
- クリック
- PV
- 検索順位
- 売上
- 成果否認
- 改善提案

## 投稿生成ルール

### すべての投稿に入れる考え方

- 最初の1文で読者の関心をつかむ
- 不安を煽りすぎない
- 「買うべき人」と「買わなくていい人」を分ける
- リンク先で最終確認するよう促す
- 広告がある場合は明示する

## 投稿ステージ

SNS投稿は、販売投稿だけにしない。以下の4段階で配分する。

| ステージ | 目的 | 初期比率 |
|---|---|---:|
| 認知投稿 | 話題に気づかせる | 30% |
| 信頼投稿 | 買わない選択肢・注意点を出す | 30% |
| 比較投稿 | 用途別・条件別に整理する | 30% |
| 行動投稿 | 比較ページへ誘導する | 10% |

初期は行動投稿を増やしすぎない。まず信頼と比較を積む。

## DRM目的ラベル

BOTは投稿ごとに、以下の目的ラベルを付ける。

- 目的
- 信用
- 問題点
- 手段
- 投資
- 行動

投稿目的が説明できないものは生成しない。

## 返信案生成ルール

将来、XやThreadsの返信案をBOTで生成する場合は、販売ではなく信頼作りに使う。

- リンクを原則入れない
- 商品名を押し付けない
- 相手を否定しない
- 1投稿1補足にする
- 煽り・宣伝・コピペ返信を禁止

### X

役割: 即時性、話題への反応、ブログへの送客。

文字数:

- 標準投稿は280字以内を前提にする
- 日本語は安全側に見て、本文は120字前後を推奨
- URL付き投稿はさらに短くする
- Premium長文投稿は初期運用では使わない

基本構成:

1. 話題の要点
2. 読者への影響
3. 買う前の判断基準
4. ブログまたは比較記事への誘導

### Threads

役割: やや会話寄りの補足、短い比較、読者の共感獲得。

基本構成:

1. ありがちな悩み
2. 判断ポイント
3. 失敗しやすい点
4. 詳細記事への誘導

### Instagram

役割: 保存される比較カード、選び方、チェックリスト。

基本構成:

1. 表紙: 買う前に見るポイント
2. 何が話題か
3. 買うべき人
4. 買わなくていい人
5. 比較ポイント
6. 注意点
7. 詳細記事への誘導

### TikTok

役割: 短尺で気づきを与える。初期は台本のみで、動画制作は後回し。

基本構成:

1. 冒頭フック
2. ありがちな失敗
3. 選ぶ基準
4. 具体例
5. 詳細はプロフィールへ

## 自動投稿禁止条件

以下に該当する場合は、投稿BOTが自動公開しない。

- 医療・治療・病気改善に見える内容
- 金融商品、投資、暗号資産の推奨
- 災害、事故、死亡、犯罪を購買誘導に使う内容
- 未成年者向けの不安訴求
- 芸能人スキャンダルや炎上
- 根拠のない健康効果
- 収益保証
- 在庫や期限を偽る内容
- 未使用なのに体験談に見える文章

## 初期投稿頻度

無料運用中は以下を上限にする。

- X: 1日1〜3投稿
- Threads: 1日1〜2投稿
- Instagram: 週2〜3カルーセル
- TikTok: 台本生成のみ、投稿は後回し

量産よりも、反応の出る型を見つけることを優先する。

## CEO確認が必要な投稿

- 広告リンク入り
- 健康、美容、フィットネス関連
- 強い比較・ランキング
- 高額商品
- サブスク契約
- 解約条件が複雑な商品
- ネガティブ比較を含む投稿

## 投稿BOTのKPI

### 初期

- 投稿生成数
- CEO承認率
- 修正率
- 重複投稿ゼロ
- NG表現検出数

### 公開後

- 表示回数
- クリック率
- 保存率
- プロフィール遷移
- ブログ流入
- アフィリエイトクリック
- 成果発生
- 否認率

## 失敗時の停止条件

次のいずれかが起きたら自動公開を止める。

- 広告表記漏れ
- 規約違反の疑い
- 同じ投稿の重複公開
- 商品情報の重大な誤り
- 健康・金融・災害系のNG突破
- APIエラー連続
- CEOが停止判断
