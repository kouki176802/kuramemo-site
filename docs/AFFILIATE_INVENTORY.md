# ASP案件在庫の運用仕様

決定日: 2026-06-25

## 目的

報酬額だけで案件を選ばず、掲載媒体、対象国、成果条件、禁止事項、リンク状態を一元管理する。公開情報で確認できない値を推測で埋めない。

原本は `data/affiliate_program_inventory.csv`。実際に提携し、リンクを取得した商品だけを `data/offers.csv` の `status=active` へ移す。

## 初期構成

### 商品網

- Amazonアソシエイト
- 楽天アフィリエイト
- もしもアフィリエイト
- A8.net
- バリューコマース

物販商品を広く扱えるが、一般に報酬率だけで大きな利益を作りにくい。比較記事の選択肢と、ニュースから商品へつなぐ在庫として使用する。

### 高単価・直接提携候補

- Adobeアフィリエイト
- Surfshark（実利用・適合時のみ）

AI・ガジェットカテゴリでは、物販だけでなくSaaS申込を収益の柱候補にする。未使用サービスの体験談や、実証していない効果は書かない。

### 海外展開用

- Amazon各国・OneLink
- impact.com
- PartnerStack

国内立ち上げ時には有効化せず、日本版で運用型を確認してから申請する。

## BOTへ投入できる条件

以下をすべて満たした案件だけを `active` にする。

1. 提携承認済み
2. 掲載するサイト・SNSをASPへ登録済み
3. 報酬と成果条件を確認済み
4. SNS掲載可否を媒体別に確認済み
5. 広告表示・禁止表現を確認済み
6. 正式なアフィリエイトURLを取得済み
7. リンク先・価格・対象地域を確認済み
8. 最終確認日を記録済み

一項目でも不明なら `research` のままとし、公開リンクを挿入しない。

## 優先順位

1. 楽天アフィリエイト: 公開SNS・ブログで開始しやすく商品網が広い
2. Amazonアソシエイト: 商品網が広いが審査・コンテンツ・誘導条件を厳守
3. もしも・A8: 個別案件から高単価候補を探す
4. Adobe: AI・クリエイティブ記事と適合する場合に申請
5. バリューコマース: サイト公開・記事整備後
6. 海外ASP: 日本版検証後

## 管理画面で入力する項目

- 正確な報酬額・料率
- 成果地点
- 否認条件
- Cookie・計測期間
- サイト掲載可否
- X・Threads・Instagram掲載可否
- メルマガ・LINE可否
- リスティング・広告配信可否
- 対象国・居住条件
- 提携開始日・終了日
- 広告素材の利用条件
- アフィリエイトURL

## 重要な禁止事項

- 未登録媒体へリンクを掲載しない
- 自動クリックやCookie付与を行わない
- 広告素材を無断加工しない
- SNS可否をASP単位で一括判断しない
- 非公開DMへ楽天リンクを送らない
- Amazonリンクへ有料広告・ブースト投稿から直接誘導しない
- 報酬額を理由に比較順位を変更しない

## 更新頻度

- 毎週: リンク切れ・案件停止
- 毎月: 報酬・成果条件・SNS可否
- 記事生成前: 商品価格・在庫・対象地域
- 規約改定通知時: 即時再確認
