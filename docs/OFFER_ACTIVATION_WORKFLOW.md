# 案件有効化ワークフロー

作成日: 2026-06-25

## 目的

ASP登録後、案件を安全に `data/offers.csv` へ追加し、BOTが記事やSNSに挿入できる状態へ進めるための手順。

初期状態では、実アフィリエイトリンクを持たない候補はすべて `research` または `pending` にする。承認前に `active` にしない。

## ステータス定義

### research

調査候補。実アフィリエイトリンクなし。

使う場面:

- 商品カテゴリだけ決まっている
- どのASPで扱うか未確定
- 報酬やSNS可否が未確認

BOTの扱い:

- 記事の題材にはできる
- 実リンクは挿入しない
- 「リンク未設定」または比較記事への内部リンクにする

### pending

提携申請中または条件確認中。

使う場面:

- ASP内で案件を見つけた
- 提携申請中
- SNS可否、成果条件、禁止表現を確認中

BOTの扱い:

- 実リンクは挿入しない
- CEO確認リストに出す

### active

掲載可能。

条件:

- 提携承認済み
- 実アフィリエイトURL取得済み
- 掲載媒体登録済み
- SNS可否確認済み
- 成果条件・否認条件確認済み
- 禁止表現確認済み
- 最終確認日記録済み

BOTの扱い:

- 記事下書きへのリンク挿入可
- SNSは媒体ごとの許可がある場合のみ可
- 高リスクカテゴリはCEO確認

### paused

一時停止。

使う場面:

- 規約変更
- 報酬条件変更
- リンク切れ
- SNS掲載条件不明
- 否認率が高い

BOTの扱い:

- 新規挿入禁止
- 既存記事の置換候補に出す

### rejected

使用不可。

使う場面:

- 提携否認
- 規約上掲載不可
- 媒体方針に合わない
- 誇大表現・高リスク案件

BOTの扱い:

- 挿入禁止
- 再提案しない

## 有効化チェックリスト

案件ごとに、次を確認する。

- [ ] ASP名
- [ ] 広告主名
- [ ] 商品・サービス名
- [ ] 対象カテゴリ
- [ ] 報酬条件
- [ ] 成果地点
- [ ] 否認条件
- [ ] Cookie期間
- [ ] 掲載可能媒体
- [ ] SNS掲載可否
- [ ] リスティング広告可否
- [ ] 禁止キーワード
- [ ] 禁止表現
- [ ] 価格・キャンペーンの扱い
- [ ] 公式URL
- [ ] アフィリエイトURL
- [ ] 最終確認日
- [ ] CEO確認

## `data/offers.csv` への入力ルール

### offer_id

英数字とアンダースコアで一意にする。

例:

- `rakuten_usb_charger_001`
- `a8_mens_skincare_brand_001`
- `adobe_express_001`

### network

ASPまたはネットワーク名。

例:

- `rakuten`
- `moshimo`
- `a8`
- `valuecommerce`
- `amazon`
- `partnerize`

### category

初期は次のどれかにする。

- `AI・ガジェット`
- `美容・フィットネス`
- `季節・暮らし`

### keywords

BOTがマッチングに使う語句。`|` 区切り。

例:

`AI|画像生成|デザイン|SNS投稿`

### problem_tags

読者の悩み。

例:

`時短|投稿作成|デザイン初心者`

### event_tags

トレンドやニュースの種類。

例:

`新機能|セール|季節需要|規格変更`

### affiliate_url

実アフィリエイトURL。`active` の場合は必須。

### landing_url

公式ページまたは商品ページ。リンク切れ確認や読者説明に使う。

### allowed_media

掲載可能媒体。`|` 区切り。

例:

`site|x|threads|instagram`

SNS可否が未確認なら `site` のみにするか、`status=pending` にする。

### status

`research`、`pending`、`active`、`paused`、`rejected` のいずれか。

## 初期に優先する案件タイプ

### 優先A

- 楽天市場の季節・暮らし用品
- 楽天市場のガジェット周辺機器
- もしも経由の物販
- Adobe ExpressなどAI・クリエイティブ系SaaS

理由:

- 記事テーマと合わせやすい
- SNSからの購買動機を作りやすい
- 初期の比較記事にしやすい

### 優先B

- メンズ美容
- プロテイン
- 家トレ用品
- 生活家電

理由:

- 需要はあるが、表現規制や誇大表現に注意が必要

### 後回し

- Amazonアソシエイト
- 転職
- 投資
- 高額スクール
- 医療寄り健康案件

理由:

- 審査や規約が厳しい
- 媒体実績が必要
- 誤表現のリスクが高い

## 登録後の作業順

1. ASPで案件を検索
2. 条件のスクリーンショットまたはメモを残す
3. `data/affiliate_program_inventory.csv` の状態を更新
4. `data/offers.csv` に `pending` で追加
5. SNS掲載可否と禁止表現を確認
6. 実アフィリエイトURLを入力
7. CEO確認
8. `active` に変更
9. BOTのdry-runでリンク挿入を確認
10. WordPress下書きで確認

## 停止条件

次の場合は即 `paused` にする。

- 広告主が終了
- リンク切れ
- SNS掲載が不可と判明
- 成果条件が大きく変更
- ステマ表記の問題がある
- 読者に誤認を与える可能性が高い
- 否認率が高い
